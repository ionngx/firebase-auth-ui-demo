Provider Bar

The Provider Bar component presents a horizontal set of buttons for each provider specified in the providers property of your configuration settings. The default for this setting is AuthProviderId.All which displays a button for all providers supported by the library. By default the Email/Password button is hidden, but it can be enabled if you are using the bar as a separate component and want to navigate to your own page to capture the email and password without using other components in the library.

Try It Out

An instance of the Provider Bar component is declared directly below this paragraph. The attributes you can use to control it are detailed below. Controls below allow you to experiment with the configuration and see the effect.

NB: At the time of writing the color attribute of ion buttons does not update when the bound value changes. Binding works to set the color when the button is created but dynamically changing it does not work.

Current Settings

Usage

This page declares the Provider Bar in the template like this

    <ionngx-firebase-auth-ui-provider-bar
      [hideEmailPasswordButton]="hideEmailPasswordButton"
      [buttonColor]="buttonColor"
      (emailPasswordSelected)="handleEmailPasswordSelected()">
    </ionngx-firebase-auth-ui-provider-bar>

Attributtes

hideEmailPasswordButton

Indicates whether the Email/Password button should be hidden

Type boolean
Default true
buttonColor

A valid Ionic theme color name

Type 'primary' | 'secondary' | 'tertiary' | 'light' | 'medium' | 'dark' | 'success' | 'warning' | 'danger'
Default primary

Events

emailPasswordSelected

Emitted when hideEmailPasswordButton is false and the user clicks on the Email/Password button. Use this even to provide your own component for capturing the email and password.

Payload void
