Sign In

The Sign In component provides a single user interface that combines inputs for Email/Password authentication and buttons for provider authentication such as Google, Facebook etc. I supports two modes bar or stack.

In bar mode the Provider Bar component is displayed below the input form.

In stack mode the Provider Stack component provides the buttons, which act responsively displaying to the right of the input form if space allows otherwise below the input form.

Try It Out

An instance of the Sign In component is declared directly below this paragraph. The attributes you can use to control it are detailed below. Use the controls below to test the effect of different settings.

mode Bar Stack

Usage

This page declares the Sign In component in the template like this

    <ionngx-firebase-auth-ui-sign-in
      [mode]="selectedMode">
    </ionngx-firebase-auth-ui-sign-in>

Attributes

mode

The mode the component should operate in. In bar mode it display the Provider Bar. In stack mode it displays the Provider Stack

Type 'bar' | 'stack'
Default bar
