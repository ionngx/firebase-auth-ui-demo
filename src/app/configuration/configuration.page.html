Configuration

During initialisation of the library you can provide a configuration object to override some or all of the configuration options. If a value is not provided during initialisation the default is used. The following options are available.

Options

enableFirestoreSync

Indicates whether user details should by synced into the Firestore database associated with the project

Type boolean
Default false
firestoreCollectionName

The name of the collection to use when synching users

Type string
Default users
guardFallbackRoute

The route to navigate to when an unauthenticated user attempts to access a protected route

Type string | string[]
Default /
guardProtectedRoutesUntilEmailIsVerified

Indicates whether to prevent access to routes protected by the built in guard until email verification is complete

Type boolean
Default true
passwordMaxLength

Maximum length allowed for passwords when creating emai/password accounts

Type number
Default 60
passwordMinLength

Minimum length allowed for passwords when creating email/password accounts

Type number
Default 8
providers

List of authentication providers supported by your app

Type AuthProviderId | AuthProviderId[]
Default AuthProviderId.All
requireEmailVerification

Indicates whether user email must be verified when creating email/password accounts

Type boolean
Default false
showToastMessageOnFailure

Indicates whether to show a toast message when an operation fails

Type boolean
Default true
showToastMessageOnSuccess

Indicates whether to show a toast message when an operation succeeds

Type boolean
Default true
signInRoute

A route to navigate to when the user initiates sign in, if not specified the sign in component will be displayed in a modal

Type string | string[]
Default undefined
signInSuccessCallback

A function to call when the user signs in successfully

Type (credential: firebase.auth.UserCredential) => void
Default undefined
signInSuccessRoute

A route to navigate to when a user successfully authenticates

Type string | string[]
Default /
signOutCallback

A function to call when the user signs out

Type () => void
Default undefined
signOutRoute

The route to navigate to when the user initiates sign out, if not specified the user will be signed out and returned to the application root

Type string | string[]
Default undefined
signUpRoute

The route to navigate to when the user initiates sign up, if not specified the sign up component will be displayed in a modal

Type string | string[]
Default undefined
signUpSuccessCallback

A function to call when the user signs up successfully

Type (credential: firebase.auth.UserCredential) => void
Default undefined
signUpSuccessRoute

A route to navigate to when a user signs up successfully

Type string | string[]
Default /
viewProfileRoute

The route to navigate to when the user initiates view profile, if not specified the user profile component will be displayed in a modal

Type string | string[]
Default undefined
stringResources

An object providing alternative strings for text displayed by the components

Type StringResources
Default See separate page
