Configuration

During initialisation of the library you can provide a configuration object to override some or all of the configuration options. If a value is not provided during initialisation the default is used. The following options are available.

Options

enableFirestoreSync

Indicates whether user details should by synced into the Firestore database associated with the project

Type boolean
Default false
guardFallbackRoute

The route to navigate to when an unauthenticated user attempts to access a protected route

Type string | string[]
Default /
signInSuccessRoute

The route to navigate to when a user successfully authenticates

Type string | string[]
Default /
guardProtectedRoutesUntilEmailIsVerified

Indicates whether to prevent access to routes protected by the built in guard until email verification is complete

Type boolean
Default true
passwordMaxLength

Maximum length allowed for passwords when creating emai/password accounts

Type number
Default 60
passwordMinLength

Minimum length allowed for passwords when creating email/password accounts

Type number
Default 8
providers

List of authentication providers support by the app

Type AuthProviderId | AuthProviderId[]
Default AuthProviderId.All
requireEmailVerification

Indicates whether user email must be verified when creating email/password accounts

Type boolean
Default false
showToastMessageOnAuthenticationFailure

Indicates whether to show a toast message when user authentication fails

Type boolean
Default true
showToastMessageOnAuthenticationSuccess

Indicates whether to show a toast message when user authentication succeeds

Type boolean
Default true
signInLabel

The label to display on the button provided to initiate signing in

Type string
Default Sign In
signInRoute

The route to navigate to when the user initiates sign in, if not specified the sign in component will be displayed in a modal

Type string | string[]
Default undefined
signOutLabel

The label to display on the button provided to initiate signing out

Type string
Default Sign Out
signOutRoute

The route to navigate to when the user initiates sign out, if not specified the user will be signed out and returned to the application root

Type string | string[]
Default undefined
signUpLabel

The label to display on the button provided to initiate signing up

Type string
Default Sign Up
signUpRoute

The route to navigate to when the user initiates sign up, if not specified the sign up component will be displayed in a modal

Type string | string[]
Default undefined
viewProfileLabel

The label to display on the button provided to initiate loading the users profile

Type string
Default Profile
viewProfileRoute

The route to navigate to when the user initiates view profile, if not specified the user profile component will be displayed in a modal

Type string | string[]
Default undefined
